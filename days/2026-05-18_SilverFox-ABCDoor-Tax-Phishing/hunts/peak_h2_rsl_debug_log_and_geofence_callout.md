# PEAK H2 — RustSL Phantom Persistence debug log + geofence resolver call-out

## Hypothesis

A host in the past seven days wrote a file named `rsl_debug.log` to a
user-writable directory and within five minutes performed an outbound DNS or
HTTP query against any of the geolocation services that RustSL uses to enforce
its country allow-list (`ip-api.com`, `ipwho.is`, `ipinfo.io`, `ipapi.co`,
`geoplugin.net`).

## Why this discriminates

The `rsl_debug.log` file is generated by the verbatim banner that the RustSL
loader writes when it registers shutdown-signal hijack (Phantom Persistence).
That banner is unique to the loader and present in every observed sample
between December 2025 and May 2026. The geofence call-out is the very first
network primitive the loader emits — before any C2 contact. Pairing both
signals discriminates RustSL from benign Rust tooling.

## Expected benign vs malicious

The debug log filename is not generic; we have never observed a benign Rust
project ship `rsl_debug.log` to user-writable directories. The geofence
services have legitimate uses (VPN clients, telemetry scripts) but the
combination with the debug-log write inside a five-minute window is
extraordinarily rare in normal operations.

## Data sources

- Defender XDR `DeviceFileEvents` (debug log write).
- Defender XDR `DeviceNetworkEvents` (geofence DNS / HTTP).
- YARA scan triggered on `rsl_debug.log` parent directory (validate the
  Phantom Persistence banner inside the file).

## KQL — paired primitives

```kql
let geo_hosts = dynamic(["ip-api.com","ipwho.is","ipinfo.io","ipapi.co","geoplugin.net"]);
let log_writes = DeviceFileEvents
| where Timestamp > ago(7d)
| where ActionType in ("FileCreated","FileRenamed")
| where FileName =~ "rsl_debug.log"
| project LogTime=Timestamp, DeviceId, DeviceName, FolderPath, InitiatingProcessFileName;
let geo_calls = DeviceNetworkEvents
| where Timestamp > ago(7d)
| where RemoteUrl has_any (geo_hosts)
| project NetTime=Timestamp, DeviceId, RemoteUrl, InitiatingProcessFileName;
log_writes
| join kind=inner (geo_calls) on DeviceId
| where NetTime between (LogTime .. (LogTime + 5m))
| project LogTime, NetTime, DeviceName, FolderPath, InitiatingProcessFileName, RemoteUrl
```

## Action on match

1. Pull the `rsl_debug.log` file. If the contents include the verbatim banner
   `God-Tier Telemetry Blinding: Deployed via HalosGate Indirect Syscalls`, this
   is a high-confidence Silver Fox RustSL hit.
2. Snapshot the host's IP-resolution cache and DNS query history (Microsoft DNS
   Analytical logs / Cloud DNS audit log).
3. Capture the loader binary and YARA-scan with
   `yara/silverfox_abcdoor_rustsl_2026.yar`. The loader uses Halo's Gate
   indirect syscalls — user-mode EDR hooks will miss `Nt*` activity. ETW-TI
   plus kernel callbacks are the only reliable in-host telemetry.
4. Re-image the host. Phantom Persistence shutdown-signal hijack survives
   reboots — uninstalling the implant requires explicit removal of the
   `RegisterApplicationRestart`-registered restart command, the run key, and
   the AppClient scheduled task before the next shutdown.
